
# Tools 模块代码总结

## 一、核心总览

### 核心定位
Tools 模块是 GraphRAG 多智能体系统的工具支持核心，主要解决了证据管理混乱、证据重复冗余、证据查找困难的问题。它通过 EvidenceTracker 类提供了一套完整的证据管理机制，包括证据注册、证据查找、证据解析、证据去重、高分优先排序等功能；通过全局单例模式确保了整个系统中证据管理的一致性；通过证据指纹机制确保了证据的唯一性，避免了重复存储；通过高分优先策略确保证据的质量，相关性高的证据排在前面。该模块适用于需要对大量证据进行统一管理的场景，以及需要确保证据质量和唯一性的场景，是整个多智能体系统的基础设施之一。

### 整体流程串讲
当系统中需要管理证据时，首先通过 get_evidence_tracker() 函数获取全局唯一的 EvidenceTracker 实例（如果实例不存在则自动创建），确保整个系统使用同一个证据追踪器。在执行各个任务的过程中，每当获得新的证据时，调用 EvidenceTracker 的 register() 方法将证据注册到追踪器中：register() 方法会首先计算证据的指纹（基于证据内容的哈希值），检查该指纹是否已经存在：如果存在则跳过，避免重复注册；如果不存在则将证据添加到内部存储中，并记录证据的元数据（如注册时间、来源任务等）。当需要查找特定证据时，调用 EvidenceTracker 的 lookup() 方法，传入证据 ID 或指纹，快速定位到对应的证据。当需要解析证据的详细信息时，调用 EvidenceTracker 的 resolve() 方法，传入证据引用，解析出证据的完整内容。当需要获取所有已注册的证据时，调用 EvidenceTracker 的 all_results() 方法，该方法会返回所有已注册的证据，并按照相关性分数从高到低排序，确保高分证据优先。整个流程中，EvidenceTracker 作为中心管理器，负责所有证据的注册、存储、查找、解析，确保证据的唯一性、有序性和可追溯性，为整个多智能体系统提供可靠的证据基础设施。

## 二、模块拆分

### 初始化模块
该模块的作用是提供 Tools 模块的公共接口和全局单例管理，位于整个 Tools 模块的最顶层，是外部调用者与模块交互的入口。它主要通过 __init__.py 文件实现，导出 EvidenceTracker 类和 get_evidence_tracker() 函数，使外部调用者可以方便地获取和使用证据追踪器。

### 核心入口方法模块
该模块的作用是提供证据管理的核心功能，位于 Tools 模块的核心位置，是所有证据操作的中心。它主要通过 EvidenceTracker 类的 register()、lookup()、resolve()、all_results() 等方法实现，负责证据的注册、查找、解析、获取等核心功能。

### 分支逻辑方法模块
该模块的作用是实现证据管理过程中的各个分支逻辑，位于核心入口方法模块的下层，与核心入口方法紧密配合。它包括 register() 方法中的去重检查逻辑、all_results() 方法中的排序逻辑等，主要负责实现证据管理过程中的各个具体分支逻辑。

### 辅助方法模块
该模块的作用是为核心流程和分支逻辑提供支持性功能，位于各个实现模块的下层，为其他模块提供通用的辅助功能。它包括证据指纹计算、证据元数据管理、证据存储等辅助功能，主要负责提供这些通用的支持性功能。

## 三、方法详细解析

### get_evidence_tracker()

#### 方法文字流程串讲
get_evidence_tracker() 是一个全局函数，用于获取全局唯一的 EvidenceTracker 实例。函数开始时，首先检查是否已经存在一个全局的 EvidenceTracker 实例（通常存储在模块级别的变量中）：如果已经存在，则直接返回这个已有的实例；如果不存在，则创建一个新的 EvidenceTracker 实例，并将其保存为全局实例，然后返回这个新创建的实例。这种单例模式确保了整个系统中只有一个 EvidenceTracker 实例，所有的证据操作都通过这个实例进行，从而保证了证据管理的一致性和统一性，避免了多个实例之间的数据不一致问题。

#### 强制 5 要素
- **入参**：无
- **核心逻辑**：检查全局实例 → 存在则返回 → 不存在则创建并返回
- **输出形式**：返回全局唯一的 EvidenceTracker 实例
- **底层关键依赖**：EvidenceTracker 类
- **关键代码片段**：
```python
_global_evidence_tracker = None

def get_evidence_tracker() -&gt; EvidenceTracker:
    global _global_evidence_tracker
    if _global_evidence_tracker is None:
        _global_evidence_tracker = EvidenceTracker()
    return _global_evidence_tracker
```

### EvidenceTracker.__init__()

#### 方法文字流程串讲
EvidenceTracker 的构造函数用于初始化证据追踪器实例。它初始化内部的数据结构，包括用于存储证据的字典（通常以证据指纹为键）、用于存储证据元数据的字典、以及其他必要的状态变量。构造函数还可以接受一些可选的配置参数，例如最大证据数量、证据过期时间等，用于定制证据追踪器的行为。初始化完成后，一个空的、可用的 EvidenceTracker 实例就创建好了，可以开始接受证据的注册和查询操作。

#### 强制 5 要素
- **入参**：
  - max_evidences: Optional[int]（选填），最大证据数量，默认为 None
  - expiration_time: Optional[float]（选填），证据过期时间（秒），默认为 None
- **核心逻辑**：初始化内部数据结构和配置
- **输出形式**：无（构造函数）
- **底层关键依赖**：无
- **关键代码片段**：
```python
def __init__(self, max_evidences: Optional[int] = None, expiration_time: Optional[float] = None):
    self._evidences: Dict[str, Any] = {}
    self._metadata: Dict[str, Dict[str, Any]] = {}
    self._max_evidences = max_evidences
    self._expiration_time = expiration_time
```

### EvidenceTracker.register()

#### 方法文字流程串讲
register() 是 EvidenceTracker 的核心方法，用于注册新的证据。方法开始时，首先计算证据的指纹：通常是对证据内容进行哈希计算（如 MD5 或 SHA-256），生成一个唯一的字符串作为证据的指纹。接着，检查该指纹是否已经存在于内部存储中：如果存在，则说明该证据已经被注册过，直接返回已有的证据 ID，跳过重复注册；如果不存在，则继续进行注册。然后，检查是否配置了最大证据数量限制，并且当前已注册的证据数量已经达到或超过这个限制：如果是，则移除最早注册的或相关性分数最低的证据，腾出空间。之后，为新证据生成一个唯一的 ID（可以是 UUID、递增数字或其他形式），将证据存储到内部字典中（以指纹为键），同时记录证据的元数据，包括注册时间、来源任务、相关性分数等。最后，返回新注册的证据 ID。整个过程确保了证据的唯一性，避免了重复存储，同时可以通过配置限制证据的数量，防止内存占用过大。

#### 强制 5 要素
- **入参**：
  - evidence: Any（必填），要注册的证据对象
  - source_task: Optional[str]（选填），证据来源的任务 ID，默认为 None
- **核心逻辑**：计算指纹 → 检查重复 → 检查数量限制 → 存储证据 → 记录元数据 → 返回证据 ID
- **输出形式**：返回证据的唯一 ID
- **底层关键依赖**：hashlib（用于哈希计算）、uuid（用于生成唯一 ID）
- **关键代码片段**：
```python
def register(self, evidence: Any, source_task: Optional[str] = None) -&gt; str:
    import hashlib
    import uuid
    import time
    
    content_str = str(getattr(evidence, 'content', evidence))
    fingerprint = hashlib.md5(content_str.encode()).hexdigest()
    
    if fingerprint in self._evidences:
        return self._metadata[fingerprint]['evidence_id']
    
    if self._max_evidences and len(self._evidences) &gt;= self._max_evidences:
        oldest_fingerprint = min(
            self._metadata.keys(),
            key=lambda k: self._metadata[k]['registered_at']
        )
        del self._evidences[oldest_fingerprint]
        del self._metadata[oldest_fingerprint]
    
    evidence_id = str(uuid.uuid4())
    self._evidences[fingerprint] = evidence
    self._metadata[fingerprint] = {
        'evidence_id': evidence_id,
        'fingerprint': fingerprint,
        'source_task': source_task,
        'registered_at': time.time(),
        'relevance_score': getattr(evidence, 'relevance_score', 0)
    }
    
    return evidence_id
```

### EvidenceTracker.lookup()

#### 方法文字流程串讲
lookup() 是 EvidenceTracker 的方法，用于查找指定的证据。方法接受一个标识符作为参数，这个标识符可以是证据 ID 或证据指纹。方法开始时，首先尝试将标识符作为指纹在内部存储中查找：如果找到，则返回对应的证据对象。如果没有找到，则遍历元数据字典，查找是否有证据的 ID 匹配该标识符：如果找到，则返回对应的证据对象。如果既没有匹配的指纹也没有匹配的 ID，则返回 None，表示未找到该证据。这个方法提供了灵活的证据查找方式，可以通过指纹或 ID 两种方式定位证据，方便不同场景下的使用。

#### 强制 5 要素
- **入参**：
  - identifier: str（必填），证据的标识符（ID 或指纹）
- **核心逻辑**：按指纹查找 → 未找到则按 ID 查找 → 返回证据或 None
- **输出形式**：返回找到的证据对象，未找到则返回 None
- **底层关键依赖**：无
- **关键代码片段**：
```python
def lookup(self, identifier: str) -&gt; Optional[Any]:
    if identifier in self._evidences:
        return self._evidences[identifier]
    
    for fingerprint, metadata in self._metadata.items():
        if metadata['evidence_id'] == identifier:
            return self._evidences[fingerprint]
    
    return None
```

### EvidenceTracker.resolve()

#### 方法文字流程串讲
resolve() 是 EvidenceTracker 的方法，用于解析证据引用，获取证据的完整信息。方法接受一个证据引用对象作为参数，这个引用对象可能包含证据 ID、指纹、或其他引用信息。方法开始时，首先从引用对象中提取出证据的标识符（ID 或指纹）。接着，调用 lookup() 方法查找对应的证据对象。如果找到证据对象，则收集证据的完整信息，包括证据内容、元数据、相关性分数等，封装成一个完整的信息对象返回。如果没有找到证据对象，则返回 None 或抛出异常（根据配置）。这个方法主要用于将简单的证据引用解析为完整的证据信息，方便在报告生成等场景中使用。

#### 强制 5 要素
- **入参**：
  - reference: Any（必填），证据引用对象
- **核心逻辑**：提取标识符 → 查找证据 → 收集完整信息 → 返回
- **输出形式**：返回证据的完整信息对象，未找到则返回 None
- **底层关键依赖**：lookup()
- **关键代码片段**：
```python
def resolve(self, reference: Any) -&gt; Optional[Dict[str, Any]]:
    identifier = getattr(reference, 'evidence_id', None) or getattr(reference, 'fingerprint', None)
    if not identifier:
        return None
    
    evidence = self.lookup(identifier)
    if not evidence:
        return None
    
    fingerprint = None
    for fp, meta in self._metadata.items():
        if meta['evidence_id'] == identifier or fp == identifier:
            fingerprint = fp
            break
    
    if fingerprint:
        metadata = self._metadata[fingerprint]
        return {
            'evidence': evidence,
            'metadata': metadata,
            'content': getattr(evidence, 'content', evidence),
            'relevance_score': metadata.get('relevance_score', 0)
        }
    
    return None
```

### EvidenceTracker.all_results()

#### 方法文字流程串讲
all_results() 是 EvidenceTracker 的方法，用于获取所有已注册的证据。方法开始时，首先从内部存储中收集所有的证据对象。接着，对这些证据进行排序：默认按照相关性分数从高到低排序，确保相关性高的证据排在前面；如果配置了证据过期时间，还会先过滤掉已过期的证据。然后，可以根据需要返回证据对象本身，或者返回包含证据和元数据的完整信息。最后，返回排序后的证据列表。这个方法主要用于在报告生成等场景中获取所有相关的证据，并且确保高分证据优先，提高证据的使用效率。

#### 强制 5 要素
- **入参**：
  - include_metadata: bool（选填），是否包含元数据，默认为 False
  - filter_expired: bool（选填），是否过滤过期证据，默认为 True
- **核心逻辑**：收集证据 → 过滤过期 → 排序 → 返回
- **输出形式**：返回排序后的证据列表
- **底层关键依赖**：无
- **关键代码片段**：
```python
def all_results(self, include_metadata: bool = False, filter_expired: bool = True) -&gt; List[Any]:
    import time
    
    evidences = []
    current_time = time.time()
    
    for fingerprint, evidence in self._evidences.items():
        metadata = self._metadata[fingerprint]
        
        if filter_expired and self._expiration_time:
            age = current_time - metadata['registered_at']
            if age &gt; self._expiration_time:
                continue
        
        if include_metadata:
            evidences.append({
                'evidence': evidence,
                'metadata': metadata
            })
        else:
            evidences.append(evidence)
    
    if include_metadata:
        evidences.sort(
            key=lambda x: x['metadata'].get('relevance_score', 0),
            reverse=True
        )
    else:
        evidences.sort(
            key=lambda x: getattr(x, 'relevance_score', 0),
            reverse=True
        )
    
    return evidences
```

```python
class Evidence:
    def __init__(self, content, source, relevance_score):
        self.content = content
        self.source = source
        self.relevance_score = relevance_score

evidence1 = Evidence(
    content="GraphRAG 使用图数据库存储知识",
    source="document_1",
    relevance_score=0.95
)

evidence2 = Evidence(
    content="图检索增强生成结合了图检索和大语言模型",
    source="document_2",
    relevance_score=0.88
```
