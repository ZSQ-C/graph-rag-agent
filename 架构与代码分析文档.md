# GraphRAG Agent 项目架构与代码分析文档

## 项目概述

GraphRAG Agent 是一个基于知识图谱的综合检索增强生成（RAG）系统，结合了 **GraphRAG** 与 **私域 Deep Search** 技术，实现可解释、可推理的智能问答系统。项目采用多 Agent 协作与知识图谱增强架构，构建完整的 RAG 智能交互解决方案。

### 核心特性
- **从零开始复现 GraphRAG**：完整实现图结构知识表示
- **DeepSearch 与 GraphRAG 创新融合**：向量数据库与知识图谱结合
- **多 Agent 协同架构**：支持不同类型 Agent 协同工作
- **完整评估系统**：提供 20+ 种评估指标
- **增量更新机制**：支持动态增量构建与智能去重
- **实体质量提升**：实体消歧和对齐机制
- **思考过程可视化**：展示 AI 推理轨迹

## 系统架构分析

### 整体架构设计

项目采用分层架构，主要分为四大模块：

1. **核心引擎层** (`graphrag_agent/`) - 智能体系统与算法实现
2. **后端服务层** (`server/`) - FastAPI RESTful API 服务
3. **前端展示层** (`frontend/`) - Streamlit 用户界面
4. **数据处理层** (`pipelines/`, `integrations/`) - 文档处理和知识图谱构建

### 核心模块架构

#### 1. 智能体系统 (agents/)

**多层级 Agent 架构**：
- **基础层**：`BaseAgent` - 提供通用功能和接口
- **功能层**：`NaiveRagAgent`, `GraphAgent`, `HybridAgent`, `DeepResearchAgent`
- **高级层**：`FusionGraphRAGAgent` - 采用 Plan-Execute-Report 多智能体编排栈

**多智能体编排栈 (multi_agent/)**：
```
planner/        # 规划器：澄清、任务分解、计划审校
executor/       # 执行器：检索、研究、反思执行器及协调器
reporter/       # 报告器：纲要生成、章节写作、一致性检查
core/           # 核心模型：PlanSpec、State、ExecutionRecord
tools/          # 工具：证据追踪、检索适配器、JSON解析
integration/    # 集成：工厂类、兼容门面
```

#### 2. 缓存管理系统 (cache_manager/)

**多层缓存架构**：
- **存储后端**：内存缓存、磁盘缓存、混合缓存
- **键策略**：简单键、上下文感知键、关键词感知键
- **向量相似度匹配**：支持语义相似的缓存查询
- **线程安全**：提供线程安全的缓存包装器

#### 3. 知识图谱构建 (graph/)

**完整建图流程**：
- **实体关系提取**：LLM 驱动的实体关系识别
- **实体处理**：消歧、对齐、合并、质量提升
- **索引管理**：文本块索引、实体索引
- **增量更新**：支持动态增量构建

#### 4. 搜索模块 (search/)

**多策略搜索体系**：
- **LocalSearch**：本地邻域搜索
- **GlobalSearch**：全局社区摘要搜索
- **HybridSearch**：混合搜索策略
- **DeepResearch**：深度推理搜索，支持思考链
- **Chain of Exploration**：知识图谱上的多步探索

#### 5. 评估系统 (evaluation/)

**20+ 种评估指标**：
- **答案质量**：EM、F1 Score、相关性、完整性
- **检索性能**：Precision、Recall、Utilization、Latency
- **图谱评估**：Entity Coverage、Graph Coverage、Community Relevance
- **LLM 评估**：Coherence、Factual Consistency、Comprehensiveness
- **深度搜索**：Reasoning Coherence、Reasoning Depth、Iterative Improvement
