# GraphRAG Agent 系统部署与使用指南

## 系统概述

GraphRAG Agent 是一个基于知识图谱的综合检索增强生成系统，提供完整的 RAG 智能交互解决方案。本指南将详细介绍系统的部署、配置和使用方法。

## 环境要求

### 硬件要求
- **内存**: 最低 8GB，推荐 16GB 或以上
- **存储**: 至少 10GB 可用空间
- **CPU**: 支持 AVX2 指令集的现代处理器

### 软件要求
- **操作系统**: Windows 10/11, macOS 10.15+, Ubuntu 18.04+
- **Python**: 3.10 或更高版本
- **Docker**: 20.10+ (可选，用于容器化部署)
- **Neo4j**: 5.22.0 (通过 Docker 部署)

## 核心功能使用指南

### 1. 基础问答功能

#### 使用 Python API
```python
from graphrag_agent.agents import FusionGraphRAGAgent

# 创建 Agent 实例
agent = FusionGraphRAGAgent()

# 基础问答
result = agent.ask("什么是优秀学生的申请条件？")
print(result)

# 带执行轨迹的问答
result_with_trace = agent.ask_with_trace("优秀学生的申请条件是什么？")
print(f"答案: {result_with_trace['answer']}")
print(f"执行轨迹: {result_with_trace['execution_log']}")

# 流式问答
async for chunk in agent.ask_stream("请详细说明奖学金评定流程"):
    print(chunk, end="", flush=True)
```

#### 使用 REST API
```bash
# 基础问答
curl -X POST "http://localhost:8000/api/query" \
     -H "Content-Type: application/json" \
     -d '{"query": "优秀学生的申请条件是什么？", "agent_type": "fusion"}'

# 流式问答
curl -N -X POST "http://localhost:8000/api/query/stream" \
     -H "Content-Type: application/json" \
     -d '{"query": "请详细说明奖学金评定流程", "agent_type": "fusion"}'
```

### 4. 评估系统使用

#### 执行系统评估
```python
from graphrag_agent.evaluation import AnswerEvaluator
from graphrag_agent.evaluation.core import AnswerEvaluationData

# 创建评估器
evaluator = AnswerEvaluator({
    'save_dir': './eval_results',
    'metrics': ['accuracy', 'relevance', 'completeness']
})

# 准备评估数据
eval_data = AnswerEvaluationData(samples=[
    {
        'question': '问题1',
        'reference_answer': '参考答案1',
        'generated_answer': '生成答案1'
    },
    # 更多样本...
])

# 执行评估
results = evaluator.evaluate(eval_data)
print(results)
```

#### 评估指标说明
项目提供 20+ 种评估指标：

**答案质量指标**:
- `accuracy`: 答案准确性
- `relevance`: 相关性
- `completeness`: 完整性
- `coherence`: 连贯性

**检索性能指标**:
- `precision`: 精确率
- `recall`: 召回率
- `utilization`: 检索结果利用率
- `latency`: 响应延迟

**图谱评估指标**:
- `entity_coverage`: 实体覆盖率
- `graph_coverage`: 图谱覆盖率
- `community_relevance`: 社区相关性

## 故障排除与支持

### 常见问题解答

**Q: 系统启动时报连接 Neo4j 失败**
A: 检查 Neo4j 服务是否正常运行，确认连接参数正确

**Q: 问答响应时间过长**
A: 优化缓存配置，减少不必要的检索，考虑升级硬件

**Q: 答案质量不理想**
A: 检查知识图谱构建质量，调整提示模板，考虑使用更高级的 Agent

**Q: 内存使用过高**
A: 调整缓存大小限制，启用磁盘缓存，优化批处理大小
